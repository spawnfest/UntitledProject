Whats the big deal ?

When your browser talks to to a web server, it uses one of "HTTP" verbs, (GET PUT POST PATCH DELETE, ETC). The HTML standard (even HTML 5) limits the browser to JSON being allowed to make these requests.

When you think about it, there isnt a good reason for HTML to not support these extended verbs, We the people say NO MORE ! We want the these verbs to be used by the hypermedia itself, connecting the verbs directly to HTML elements.

This may seem a bit foreign the first time you encounter this, how can you have this kind of power. Who would I want this ? Buckle up because its a bumpy ride.


Motiviation
  1. Why should only a and form be able to make HTTP requests?
  2. Why should only click & submit events trigger them?
  3. Why should only GET & POST methods be available?
  4. Why should you only be able to replace the entire screen?

By removing these arbitrary constraints, htmx completes HTML as hypertext.

Why does this matter ? Because almost all of the actions you want to do from the web client to the server are http verbs, previous locked up behind javascript and json.
